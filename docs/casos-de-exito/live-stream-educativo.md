# Live stream educativo — Atlas de casos de éxito

> Fichas de creators de la modalidad `live-stream-educativo`. **Nota de método:** análisis general (verificado 2026-04-19). No observación VTT (el formato live hace VTT auto-subs menos útil — mucha conversación no transcribe limpio).
>
> **Modalidad:** `live-stream-educativo` — ejes default: arco performativo · objetivo adquirir-habilidad · edición zero · guión improvisado · rol demostrador. Ver [docs/arquitectura/modalidades-y-ejes.md](../arquitectura/modalidades-y-ejes.md).
>
> **Particularidad del formato:** `grado-edicion: zero` colapsa la mitad de las decisiones de post-producción. La enseñanza ocurre en tiempo real, con errores, reinicios y tangentes; la "retención" se mide por horas-sesión y engagement sostenido en chat, no por APV.

**Nota cruzada:** ThePrimeagen (ficheado en [tutoriales-tecnicos.md](tutoriales-tecnicos.md#theprimeagen-p2-ficha-theprimeagen)) opera en este modo en sus streams diarios de 4-6h. La ficha principal está allí; el modo live es una dimensión del canal, no un creator separado.

## Casey Muratori — Handmade Hero [P2-ficha-casey-muratori]

- **Canal:** https://www.youtube.com/@MollyRocket + Twitch. Sitio: https://handmadehero.org
- **Nicho específico:** live coding de game engine completo en C desde cero, sin dependencias. Curso "Performance-Aware Programming" paralelo. Un canon para programadores que quieren entender cómo funcionan las cosas a bajo nivel.
- **Escala:** ~80K subs YT, Discord comunitario activo, Patreon. No-top-charts, pero autoridad culta profunda en el nicho.
- **Duración típica:** 1-3h por stream. Handmade Hero es serie de 600+ episodios; "Performance-Aware Programming" es curso estructurado.
- **Hook pattern:** estado del día + goal concreto para la sesión ("Today we're going to implement X") + recap breve del contexto anterior.
- **Estructura típica:** diagnóstico del problema → diseño a voz alta → implementación incremental → debugging → comentario sobre decisiones. Muratori explica mientras codifica; el *modo de pensar* es el contenido.
- **Visual style:** pantalla de código (Emacs / terminal) a pantalla completa + cámara pequeña de Muratori + chat lateral en el stream de Twitch. En el re-upload a YouTube la cámara a veces se omite.
- **Scripting:** improvisado (con prep técnico previo — Muratori investiga sistema, arquitectura, y planea el día). El código en sí es improvisado.
- **Rol del presentador:** `demostrador`. Voz pedagógica con autoridad técnica visible; sarcasmo seco como rasgo.
- **CTA / cierre:** suscribe + Patreon + curso pagado (handmadehero.org).
- **Ejes reales:** arco performativo · objetivo adquirir-habilidad · edición zero (Twitch) / light (YouTube re-upload) · guión improvisado · rol demostrador.
- **Videos de referencia:** "Day 001: Setting Up the Windows Build" (Handmade Hero), curso "Performance-Aware Programming".
- **Lo que NO copiar:** Muratori tiene 20+ años como programador profesional (ex-RAD Game Tools) y la credibilidad técnica es pre-requisito visible. El formato "live coding serio de 600+ episodios" requiere sostener el interés propio durante años — la motivación de Muratori se ve en cada sesión. Un creador nuevo replicando el formato sin oficio técnico queda como instructor dudoso.

**Verificado:** 2026-04-19 (análisis general).

## Tsoding (Alexey Kutepov) [P2-ficha-tsoding]

- **Canal:** https://www.youtube.com/@TsodingDaily + Twitch.
- **Nicho específico:** live programming en Haskell, C, compiladores desde cero, lenguajes de programación experimentales. Estilo académico-irónico.
- **Escala:** ~400K subs YT. Comunidad culta en programación de sistemas.
- **Duración típica:** 3-5h por stream; re-uploads a YouTube frecuentemente en 1-2h editados.
- **Hook pattern:** explora el problema desde cero, voz bromista-irónica sobre el estado actual del lenguaje/herramienta.
- **Estructura típica:** abstracción primero → implementación → errores intencionales como enseñanza. Tsoding a veces implementa cosas mal deliberadamente y luego corrige, para mostrar el razonamiento detrás.
- **Visual style:** pantalla de código + face-cam + chat. Edición para YouTube es light (cortes naturales del stream).
- **Scripting:** improvisado. Prep muy ligero; el descubrimiento en vivo es parte del valor.
- **Rol del presentador:** `demostrador` con voz académico-irónica marcada.
- **CTA / cierre:** subscribe + GitHub + Patreon.
- **Ejes reales:** arco performativo · objetivo adquirir-habilidad · edición zero (Twitch) / light (YT) · guión improvisado · rol demostrador.
- **Videos de referencia:** serie "Porth: A Concatenative Programming Language", "I'm Writing a Programming Language".
- **Lo que NO copiar:** la voz académico-irónica es estilo personal inimitable. La decisión de hacer "cosas mal primero" sólo funciona con audiencia técnica que puede seguir el razonamiento — con audiencia principiante produce confusión (viola pilar 1 §6.1 — misconception handling sin framing explícito).

**Verificado:** 2026-04-19 (análisis general).
